# Contributing to Playbook

Thanks for helping improve Playbook.

## Development Setup

Prerequisites:

- Node.js >= 22
- pnpm >= 10

Install and build:

```bash
pnpm install
pnpm build
pnpm test
```

## Typical Workflow

1. Create a branch for your change.
2. Keep changes focused and aligned with Playbook governance.
3. Run `pnpm build` and `pnpm test` before opening a pull request.
4. Do not update README for new rules or commands; rely on CLI discovery (`playbook --help`, `pnpm playbook rules`, `pnpm playbook explain <id>`).

## Governance Audit Habit (post-merge follow-up)

After landing architecture/governance-adjacent changes, run these non-blocking checks to keep governance visible and healthy without immediately widening CI enforcement:

- `pnpm playbook audit architecture` — run when architecture contracts, deterministic boundaries, or guardrail checks are touched.
- `pnpm playbook doctor` — run before handoff/review to surface a lightweight cross-signal health summary.
- `pnpm playbook docs audit` — run when docs, governance workflows, or roadmap-linked expectations are updated.

These checks are intentionally habit-forming workflow guidance first; broaden enforcement only for low-false-positive deterministic checks.

## Project Areas

- `packages/cli` — command surface and template scaffolding.
- `packages/engine` — analysis and rule verification engine.
- `templates/repo` — files generated by `pnpm playbook init`.
- `docs` — governance, architecture, roadmap, and onboarding documentation.

For additional contributor setup notes, see [`docs/contributing.md`](docs/contributing.md).
