# Minimal Reasoning Engine

## Purpose

Define the smallest complete reasoning-engine architecture Playbook must prove before broadening into generalized cross-repository automation.

Minimal complete loop:

`observe -> atomize -> connect -> compress -> govern -> enforce -> reflect`

This loop is complete only when outputs from `reflect` can inform the next `observe` cycle through explicit, governed proposals.

## Seven minimum layers

1. **Evidence intake**
   - deterministic ingestion from repo/runtime artifacts (verify, plan, apply, docs, analyze)
   - no inferred evidence without artifact lineage
2. **Atomic memory**
   - zettels with evidence refs and deterministic IDs
   - stable shape for retrieval and comparison
3. **Structural memory**
   - typed graph + deterministic groups over linked zettels
   - evidence relationship substrate for synthesis
4. **Compression**
   - candidate pattern synthesis and draft pattern card generation
   - deterministic bucketing/compaction before promotion
5. **Governance**
   - explicit review queue and promotion decisions
   - append-only decision artifacts
6. **Contract proposal layer**
   - generate contract mutation proposals only
   - no direct contract mutation in the minimal version
7. **Meta introspection**
   - analyze reasoning artifacts and emit bounded improvement findings/proposals
   - advisory outputs only; no autonomous doctrine mutation

## Minimal out-of-scope boundary

The minimal version must explicitly exclude:

- multi-repo orchestration
- broad functor ecosystem
- advanced topology compression
- autonomous doctrine mutation
- generalized UI platform

These remain post-proof expansion surfaces.

## Controlled rollout path

### Phase 1: self-host on Playbook

Prove the full loop against Playbook's own deterministic artifacts and governance surfaces.

### Phase 2: bounded pilot on Fawxzzy Fitness

Run the same loop in a constrained pilot with explicit inclusion/exclusion boundaries.

### Phase 3: controlled cross-repo transfer

Only after Phases 1 and 2 are stable, expand to selective cross-repo transfer with strict governance gates.

## Acceptance criteria: minimal reasoning engine proven

Minimal reasoning engine is proven when Playbook can:

- ingest deterministic repo artifacts
- produce zettels with evidence refs
- build deterministic graph/group artifacts
- synthesize draft pattern cards
- support explicit promotion decisions
- generate contract mutation proposals without direct mutation
- analyze its own artifacts and emit improvement findings

## Fawxzzy Fitness pilot guidance

Initial include set:

- verify outputs
- zettels
- graph/grouping
- candidate patterns
- draft pattern cards
- review queue

Initial exclusion set:

- automatic contract mutation
- automatic doctrine tuning
- cross-repo pattern propagation

## Governance additions

Rule:
Prove one complete closed-loop reasoning engine before expanding scope to generalized cross-domain automation.

Pattern:
A strong pilot repo improves both the product being built and the reasoning engine observing it.

Failure Mode:
Using a second repo too early without bounded rollout turns a proving ground into a noise amplifier.
