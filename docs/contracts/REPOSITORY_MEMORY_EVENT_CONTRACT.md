# Repository Memory Event Contract

## Purpose

Playbook episodic memory captures high-signal command outcomes as append-oriented runtime artifacts under `.playbook/memory/`.

This contract does **not** alter the structural graph contract (`.playbook/repo-graph.json`) and memory artifacts are **not** regenerated by `playbook index`.

## Storage model

- Event artifacts: `.playbook/memory/events/<eventInstanceId>.json`
- Lookup index: `.playbook/memory/index.json`

Memory artifacts are local runtime state by default and intended to be gitignore-oriented.

## Event schema (`schemaVersion: "1.0"`)

Each event includes:

- `schemaVersion`
- `kind` (`verify_run`, `plan_run`, `apply_run`, `pr_analysis`, `failure_ingest`)
- `eventInstanceId`
- `eventFingerprint`
- `createdAt`
- `repoRevision`
- `sources`
- `subjectModules`
- `ruleIds`
- `riskSummary`
- `outcome`
- `salienceInputs`

### `eventInstanceId` vs `eventFingerprint`

- `eventInstanceId`: unique capture-instance identity (append-oriented runtime identity).
- `eventFingerprint`: deterministic semantic identity derived from normalized event content; used for dedupe/clustering across runs.

## Index schema (`schemaVersion: "1.0"`)

`index.json` is keyed for deterministic lookups by:

- `byModule`: module -> event refs
- `byRule`: rule id -> event refs
- `byFingerprint`: semantic fingerprint -> event refs

Each event ref is a repository-relative memory path (for example `events/verify_run-<uuid>.json`).
